## Architecture

The pipeline is: **CSV row → `LabelRenderer.render()` → PNG file**.

```
cli.py          Entry point — fail-fast validation, iterates records, calls renderer
config.py       Loads layout.json (strips _meta keys)
data_loader.py  Reads CSV/Excel via pandas; validates that CSV columns match layout keys
renderer.py     LabelRenderer — core rendering logic (text + barcode paste onto template copy)
barcode_gen.py  normalize_jan() + render_barcode() — EAN-13 via python-barcode ImageWriter
```

**Key design invariant**: all pixel coordinates and field specs live in `config/layout.json`. Swapping the template image only requires updating `template.png` and `layout.json`; no code changes needed.

### layout.json field types

- `"type": "text"` — rendered via PIL `draw.text()` with native `anchor` support; `max_width` triggers per-character wrap (≤2 lines, truncated with `…`)
- `"type": "barcode"` — EAN-13 PNG generated by `python-barcode`, resized, optionally rotated, then pasted; anchor is resolved manually to a top-left paste coordinate since `Image.paste()` doesn't support anchors

### Barcode rendering flow

1. `normalize_jan()` — accepts 12-digit (auto check digit) or 13-digit (validates check digit)
2. `render_barcode()` — generates EAN-13 PNG with `ImageWriter`, resizes to `width × height` from spec
3. `show_text=true` adds digit overlay using evenly-spaced PIL text characters
4. Applied rotation via `img.rotate(rotation, expand=True)`, then anchor-to-paste-xy translation

### Font handling

Fonts are loaded from `fonts/NotoSansCJK-{Regular,Bold}.otf` — **never system fonts**. `LabelRenderer._font()` and `._bold_font()` cache by size via `@lru_cache`. Use `font.getbbox(text)` for width measurement, not the deprecated `font.getsize`.

### Error handling contract

- Missing template/font/layout at startup → `typer.Exit(1)` (fail fast)
- Missing CSV columns at startup → report all missing at once, `typer.Exit(1)`
- Invalid JAN per row → skip that row, collect in `failed` list, report at end, `typer.Exit(2)`
- Empty text field → silently skip (allows optional fields)
